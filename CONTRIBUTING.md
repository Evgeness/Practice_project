# Contribution workflow

1. Создать issue или выбрать задачу.
2. Обновить `main`.
3. Создать role-prefixed feature branch.
4. Делать небольшие Conventional Commits.
5. Запустить локальные тесты и линтеры.
6. Rebase на свежий `origin/main`.
7. Push ветки и Pull Request.
8. Исправить замечания review отдельными коммитами или аккуратным squash перед merge.
9. После merge удалить ветку.

Полный командный порядок: `docs/TEAM_GIT_PLAN.md`.
