# План тестирования

## Unit

- валидация расширения, размера и пустого файла;
- PDF/DOCX parsing;
- чанкинг и overlap;
- создание русского анализатора;
- преобразование результатов Elasticsearch;
- Redis round-trip;
- загрузка API с fake DB/ES;
- cache hit поиска;
- проверка логина/пароля;
- создание, проверка, изменение и истечение токена;
- отклонение запросов без Bearer-токена.

## E2E

Критический сценарий:

```text
открыть UI → войти → загрузить sample.docx → дождаться Готово →
ввести запрос → получить карточку результата
```

## Нагрузочное тестирование

Locust сначала авторизует каждого виртуального пользователя. Цель: 50 одновременных пользователей в течение 2 минут.

Команда:

```bash
locust -f load-tests/locustfile.py --host http://localhost:8000 \
  --headless -u 50 -r 5 -t 2m --csv=report
```

В отчет внести:

- число запросов;
- число ошибок;
- median;
- p95;
- среднее время ответа;
- сравнение cache hit и cache miss.

## Precision@3

Подготовить минимум 10 запросов. Для каждого указать ожидаемый документ. Запустить `scripts/evaluate_precision.py` и приложить JSON-результат.

## Критерии готовности

- unit coverage не ниже 50%;
- E2E critical flow проходит;
- нет HTTP 5xx при валидной нагрузке;
- нужный документ входит в top-3 для большинства эталонных запросов;
- приложение запускается с чистыми volumes одной командой.
