# Руководство пользователя

## Запуск

1. Установите Docker Desktop или Docker Engine с Compose.
2. Скопируйте `.env.example` в `.env`.
3. Выполните `docker compose up --build -d`.
4. Откройте `http://localhost:3000`.

## Регистрация и вход

Для новой учётной записи:

1. Откройте вкладку «Регистрация».
2. Укажите логин длиной от 3 символов.
3. Укажите пароль длиной от 8 символов и повторите его.
4. Нажмите «Зарегистрироваться».
5. После успешной регистрации система автоматически выполнит вход.

Логин должен быть уникальным. Регистр букв при проверке уникальности не учитывается: `Student` и `student` считаются одним логином.

Для обычного входа откройте вкладку «Вход», введите логин и пароль и нажмите «Войти».

Демонстрационные данные по умолчанию:

```text
Логин: student
Пароль: practice2026
```

Они создаются при первом запуске из `AUTH_USERNAME` и `AUTH_PASSWORD` в `.env`. Пароли зарегистрированных пользователей сохраняются в PostgreSQL только в виде защищённых хешей.

Для завершения сеанса нажмите «Выйти». При истечении токена система автоматически вернёт страницу входа.

## Загрузка документов

1. Перетащите PDF/DOCX в область загрузки либо нажмите на неё.
2. Дождитесь состояний «Загрузка...», «Индексация...», «Готово».
3. Файл появится в таблице.

Ограничения: PDF/DOCX, до 20 МБ, нужен текстовый слой.

## Поиск

1. Введите фразу.
2. Нажмите Enter или «Найти».
3. Просмотрите имя файла, страницу, фрагмент и релевантность.
4. Совпадения выделяются жёлтым.
5. Для перехода используйте кнопки пагинации.

## Частые ошибки

- «Неверный логин или пароль» — проверьте введённые данные.
- «Пользователь с таким логином уже существует» — выберите другой логин.
- «Требуется авторизация» — войдите заново.
- «Срок действия токена истёк» — повторите вход.
- «Разрешены только файлы PDF и DOCX» — выбран неверный формат.
- «Размер файла превышает 20 МБ» — уменьшите файл.
- «В документе не найден текст» — вероятно, это скан без OCR.
